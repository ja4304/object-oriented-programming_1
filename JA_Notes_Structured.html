Use Functions

Object Oriented Programming

  • Object Oriented Programming is a method that is organized around objects and data rather than procedures and logic.
  • This differs from other views which treat programs as a logical procedure that takes input data, processes it, and produces and output.
  • This approach helps when dealing with large projects because you can break it into pieces(objects) and work in sections.
  • One way OOP is similar to HTML is the use of classes. In HTML, classes can be used to apply styling and formatting to all tags that are marked with that class. In Python, classes can be used to apply rules and methods to all instances of that class.
  • One of the principles of OOP is the concept of Abstraction. By using abstraction, the programmer hides all but the relevant data about an object in order to reduce complexity and increase efficiency. He/she focuses only on the details that are important for his/her purpose.

When Functions Do Not Suffice - Using Classes

  • A class is like the blue print of a building. It contains some general rules and attributes of the module
  • When using a class, you create an Instance or Object that contains the specific details for that use
  • A subclass can be defined and related to the initial class. It will inherit the definitions from the class. This is called Inheritance
  • An example: Vehicle could be a parent class. All vehicles would have similar characteristics. Truck or Car could be subclasses, or child classes, related to the superclass, Vehicles.
  • These child classes would inherit some or all of their characteristics from the class(wheels, doors, etc), but could also contain features that are different(body style, # of passengers, etc).
  • An Object is a specific instance of a subclass or class. An example would be a Ford_F150. It would contain attributes of the Truck subclass, and the Vehicle class.
  • Each object can contain procedures that are called methods. A method in an object can only access the data known to that object, which ensures data integrity among a set of objects.
  • From this organization, we can also have Class Variables, which belong to the class as a whole, and Instance Variables which belong to specific objects.
  • Calling a class creates space in memory for the instance that did not exist before
  • Classes like turtle can create geometric shapes
  • There are external Python packages such as twilio that can be used to complete tasks.
  • Functions like Open are examples of Built-in Functions. These are functions that are commonly used and always available. They are located in the Python Standard Library
  • The Python Standard Library contains built-in modules that provide access to system functionality that is commonly needed. This resource saves the programmer from having to create solutions for common functions in programming.
